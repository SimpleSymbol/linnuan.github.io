# 聆暖时光 · 云端 Demo 部署说明

本包为「聆暖时光」完整 Demo(前后端 + 最新暖橙配色),可部署到支持 Node.js 的云服务。

## 包内结构

```
聆暖时光-cloud-demo/
├── h5/                  # 前端构建产物（静态文件，直接托管）
│   ├── index.html
│   ├── assets/
│   └── .nojekyll
├── server/              # 后端源码（Node.js + Express）
│   ├── main.js
│   ├── package.json
│   └── .env.example     # 环境变量模板（需复制为 .env 并填 key）
└── 部署说明.md          # 本文件
```

---

## 一、后端部署(server/)

后端是 Node.js + Express,负责 AI 对话、回忆录生成、登录、社区等。

### 1. 上传 server/ 到云服务器
把 `server/` 目录上传到你的云服务器(腾讯云 CloudBase 云托管 / 云服务器 / 任何支持 Node 的平台)。

### 2. 安装依赖
```bash
cd server
npm install
```

### 3. 配置环境变量
把 `.env.example` 复制为 `.env`,填入你的 DeepSeek API Key:
```bash
cp .env.example .env
# 编辑 .env，填入：
# PORT=3000
# DEEPSEEK_API_KEY=sk-你的key
```
> DeepSeek Key 获取:https://platform.deepseek.com/

### 4. 启动
```bash
npm start
# 或开发模式
npm run dev
```
启动后会监听 3000 端口,看到 `✅ 服务器运行在 http://localhost:3000` 即成功。

### 5. (云托管特有)容器化
若用腾讯云 CloudBase 云托管,在 server/ 下加一个 Dockerfile:
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

## 二、前端部署(h5/)

`h5/` 是纯静态文件,托管到任意静态服务即可。

### 方式 A:同域部署(推荐,零跨域)
把 h5/ 和后端放同一域名,前端用 `/api` 同源访问,无需任何配置。

Nginx 示例(前端静态 + 反代后端):
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/lingnuan-h5;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反代到 Node 服务
    location /api/ {
        proxy_pass http://127.0.0.1:3000/api/;
        proxy_set_header Host $host;
    }
}
```

### 方式 B:跨域部署(前后端不同域名)
编辑 `h5/index.html`,在 `<head>` 里加:
```html
<script>
  window.__APP_CONFIG__ = { API_BASE: 'https://你的后端域名/api' };
</script>
```
> 后端已开启 CORS,跨域请求可正常发出。
> ⚠ 若站点是 HTTPS,后端也必须 HTTPS,否则浏览器拦截 mixed-content。

## 五、核心能力

- **AI 情感聊天**:DeepSeek 驱动,温暖陪伴式对话

## 六、注意事项

1. **DeepSeek Key 安全**:`.env` 不要提交到公开仓库,Key 泄露会被盗刷
2. **TUICallKit 密钥**:`debug/GenerateTestUserSig.js` 含测试密钥,生产环境务必迁移到服务端生成 UserSig
3. **HTTPS**:线上建议全站 HTTPS,后端 HTTP 会被浏览器拦截
4. **数据存储**:当前用文件存储(JSON/Markdown),demo 够用;若用户量大建议换数据库
