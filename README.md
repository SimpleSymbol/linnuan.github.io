# 聆暖时光 — AI 情感陪伴与数字回忆录平台

> **让每一段独处都有温度，让每一份记忆都有归处。**

---

## 项目概览

**聆暖时光** 是一款面向老年群体与需要情感陪伴人群的智能应用，融合了 **AI 情感对话**、**数字回忆录**、**情感监测** 与 **志愿者关怀** 四大核心模块。通过大语言模型（DeepSeek）与人性化交互设计，为用户打造一个有温度、有记忆、有回应的陪伴空间。

| | |
|---|---|
| **技术栈** | uni-app 3.0 (Vue 3) + Vite + Express + DeepSeek API |
| **跨端支持** | Web / Android / iOS / 微信小程序 / 支付宝小程序 |
| **后端引擎** | Node.js Express，文件级持久化存储 |
| **项目版本** | v2.0.0 |

## 核心功能

### 💬 AI 情感陪伴

基于 DeepSeek 大语言模型，构建**多轮对话**的智能陪伴助手：

- **温暖开场**：每次新会话自动生成贴心问候，降低使用门槛
- **上下文记忆**：对话持久化到文件，同一会话内 AI 记住上下文，让聊天更自然连贯
- **会话管理**：支持查看历史会话、创建新对话、删除旧会话，侧边栏一键切换

### 📖 智能回忆录

引导用户回溯人生故事，AI 辅助整理成文：

- **AI 引导式对话**：以温和提问引导用户讲述往昔，自动收集素材
- **一键生成回忆录**：AI 将零散对话整理为结构完整、语言优美的叙事文章
- **预览与修改**：支持对已生成回忆录进行二次编辑和优化
- **历史存档**：所有回忆录归档，方便重温与分享

### 📊 情绪监测与分析

洞察用户两日内的情绪波动，提供智能关怀建议：

- **48 小时窗口分析**：自动读取近期所有对话记录，通过 AI 综合分析情绪状态
- **七种情绪识别**：开心😊、平和😌、孤单🥺、难过😢、思念家人🏠、烦躁😤、焦虑😰
- **智能建议**：根据当前情绪状态生成针对性建议（如推荐听戏曲、联系子女等）
- **趋势追踪**：记录历史情绪变化，帮助监护人或志愿者了解用户心理动态

### 🤝 志愿者系统

搭建用户与志愿者之间的温情桥梁：

- **视频通话**：基于腾讯云 TUICallKit 的一键视频呼叫，连接志愿者
- **社区互动**：发布动态、互动点赞，构建线上温暖社群
- **管理后台**：志愿者可查看用户列表、管理个人资料

```

│  └──────────┘ └──────────┘ └──────────────────┘ │
│              uni-app (Vue 3)                     │

├─────────────────────────────────────────────────┤
│                  服务端层                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ AI 对话   │ │ 回忆录   │ │  情感分析         │ │
│  │ 引擎      │ │ 创作引擎  │ │  引擎             │ │
│  ├──────────┤ ├──────────┤ ├──────────────────┤ │
│  │ 会话管理  │ │ 用户认证  │ │  社区/志愿者      │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
│              Express.js                          │
├─────────────────────────────────────────────────┤
│              存储层                               │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ 会话文件  │ │ 回忆录文件│ │  用户/社区数据     │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
│         JSON 文件系统（加锁安全写入）              │
└─────────────────────────────────────────────────┘
```

### 关键设计亮点

| 设计 | 说明 |
|------|------|
| **并发写锁** | 基于 `async-mutex` 实现文件级互斥锁，防止并发写入数据错乱 |
| **双格式对齐** | 前端展示格式与后端存储格式分离，通过 `sessionMessages` 桥接上下文 |
| **情感分析管线** | 48h 会话筛选 → 内容截断 → AI 情感推断 → 结构化输出（mood + suggest） |
| **Session 生命周期** | 创建 → 对话持久化 → 加载历史 → 删除，全链路闭环 |
